(For more detailed release notes, see https://github.com/fredrikr/nail/releases )

Upcoming release:

- Made build_dictionary.rb a lot faster (3+ times faster for a test scenario).
- Fixed bug: OpenSub and CloseSub in grammar.inf didn't set scope_modified = true.
- New feature: An input line starting with "*" is treated as a comment (typically used to make remarks in a transcript).
- Changed ItOrThem routine in parser.h, so it prints (name) player, instead of "yourself", to if a  game changes the name of the player object, it's picked up here as well. 

===
v1.4, 2026-07-09

- Added System_file directive in parser.h (without this, game authors can't replace routines in this file).
- Fixed typo in grammar.inf: 'transscript' => 'transcript'
- Fixed bug in grammar.inf: AfterRoutines was run twice for Look action.
- Made the default Examine message (in grammar.h) a proper message, MSG_EXAMINE_NOTHING_SPECIAL.
- bin/build_dictionary.rb now takes two optional parameters -s and -e, to decide which range of numbers to try as a factor, e.g. build_dictionary.rb -s 2 -e 100 dict.inf game.inf 
- Fixed scope bug in nail.h: parts of a transparent, inanimate object were not added to scope. (E.g. a button object "in" a machine object).
- Fixed bug in TakeSub in grammar.inf: Player could take things that were held by an actor or were part of an inanimate object.
- Fixed bug in nail.h, procedures _InitObjects and MoveFloatingObjects: They were using the floating_objects array as a word array, but it's declared as a byte array. This would cause a crash when the array was more than half-full.

===
v1.3, 2026-07-05

- Instead of failing with an error message, build_dictionary.rb will now make up names (W_SPECIAL_WORD_n)for dictionary words which contain special characters, or where the constant name is already in use for another dictionary word.
- Library doesn't keep running each_turn or timers/daemons if deadflag is set.
- Routine RunEntryPointRoutine wasn't implemented, nor needed. All calls to RunEntryPointRoutine in nail.h and the single call in Look in grammar.inf have been changed to just call the entry point routine directly instead.
- Moved nail.h and parser.h to new folder /lib. Changed build scripts to tell compiler about the library path, and to expect parser.h to be in /lib . Changed game.inf in template and cloak to add ">" to name of files that are located in game directory. Changed bin/transform_code.rb to be able to read parser.h from /lib but write generated_parser.h to game directory.

===
v1.2.1, 2026-06-30

Bugfixes:

- Changed order of files parser.h and grammar.inf in build scripts
- Updated generate_dictionary.rb to require word constant definitions to appear last on a line.

v1.2, 2026-06-30

New/changed functionality:

- Added support for typing multiple commands on a line, separated by ".".
- Moved newline before prompt from nail.h to message MSG_PROMPT.
- Moved declarations of word constant names needed by NAIL's standard definitions from game.inf to parser.h, and added constant for './/' (Authors can still add their own word constants at the beginning of game.inf)
